# 简历版本管理



## 概述

你应该有简历，你应该优化你的简历，你应该针对不同岗位去定制你的简历。

如此一来，随着求职的进行，你会拥有大量不同的简历。

通过`文件系统`管理这些简历绝对是一种噩梦。

因此我们需要引入`版本控制系统`，最著名的莫过于`git`。

通过`git`管理简历不仅简单高效，同时还利于向`AI、他人`请求简历指导，以及应用它们。

因为`AI/他人`将充分了解你的简历的完整的"生命周期"，而不是一个"扁平"的`pdf文件`。



## 工具

- git
- vscode或其fork（cursor、windsurf、trae...）：用于简历文本的编辑，以及提供git客户端
- github：用于向`AI、他人`共享简历的版本信息



## 规范

- 仓库：储存你的简历
- 仓库分支
  - `main`：你的**基本简历**，没有进行优化和定制
  - `opti/*`：**优化后的简历**，可以使用`opti`，也可以使用`opti/*`进行进一步划分不同版本（比如面向不同公司规模、面向不同岗位方向）
  - `job/*`：**定制后，用于投递特定岗位的简历**，如`job/bytedance-frontend-*`（公司-方向-职位）
  - `job/*/interview`：**拿到面试的简历**，来自某个`job/*`
  - `job/*/offer`：**拿到offer的简历**，来自某个`job/*`
